# Reviewer Response

Use this skill to turn review comments into a calm revision plan and a
point-by-point response.

## Workflow

1. Classify comments.
   - Major validity concern
   - Missing experiment or analysis
   - Writing/clarity issue
   - Citation/background request
   - Formatting or policy issue
   - Misunderstanding caused by presentation

2. Decide the response strategy.
   - Accept and revise when the reviewer is right.
   - Clarify when the work already addresses the concern but the paper did not
     make it visible.
   - Add evidence when a claim needs stronger support.
   - Narrow the claim when the requested evidence is not available.
   - Push back only when the requested change is incorrect, infeasible, or
     outside scope, and explain why politely.

3. Draft the response.
   - Start with a short acknowledgement.
   - State the action taken.
   - Point to the revised section, figure, table, or appendix.
   - Explain remaining scope limits in clear professional language.

4. Check consistency.
   - Ensure every promised change exists in the manuscript.
   - Ensure numbering and section references match the revised draft.
   - Do not claim new experiments, statistics, or citations were added unless
     they actually were.

## Tone

Use concise, professional prose. Avoid flattery, argument loops, and apology
clusters. The response should make the revision easier to verify.
